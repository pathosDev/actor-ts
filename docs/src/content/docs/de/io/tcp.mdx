`TcpSocketActor` umschließt eine **clientseitige** TCP-Verbindung.
Zum **Lauschen** auf einem Port (eingehende TCP-Verbindungen
annehmen) lautet das Framework-Muster: "spawne pro eingehender
Verbindung von einem Listener-Actor genau einen Client-Actor".

```ts
import { ActorSystem, Props, TcpSocketActor, TcpSocketOptions } from 'actor-ts';

const tcpSocketOptions = TcpSocketOptions.create()
  .withHost('metrics-collector.example.com')
  .withPort(8125);
const tcp = system.spawn(
  Props.create(() => new TcpSocketActor(
    tcpSocketOptions,
  )),
  'tcp-client',
);

// Rohe Bytes senden:
tcp.tell({ kind: 'send', payload: new Uint8Array([0x01, 0x02, 0x03]) });

// Oder Text:
tcp.tell({ kind: 'send-text', payload: 'PING\n' });

// Eingehende Bytes subscribieren:
tcp.tell({ kind: 'subscribe', subscriber: protocolHandler });
```

```ts
interface TcpSocketOptionsType extends BrokerCommonOptionsType {

```

## Eingehende Nachrichten

```ts
import { match } from 'ts-pattern';

class ProtocolHandler extends Actor<TcpInbound> {
  override onReceive(message: TcpInbound): void {
    match(message)
      .with({ kind: 'data' },         (m) => this.onData(m))
      .with({ kind: 'connected' },    ()  => this.onConnected())
      .with({ kind: 'disconnected' }, ()  => this.onDisconnected())
      .exhaustive();
  }

  private onData(message: Extract<TcpInbound, { kind: 'data' }>): void {
    this.handleBytes(message.payload);   // Uint8Array
  }

  private onConnected(): void {
    this.log.info('TCP verbunden');
  }

  private onDisconnected(): void {
    this.log.info('TCP getrennt');
  }
}
```

Jede `data`-Nachricht trägt einen Chunk Bytes — **KEINE** logische
Nachricht.  TCP ist ein Byte-Stream; Framing ist dein Job.

## Framing

```ts
import { TcpSocketActor, TcpSocketOptions } from 'actor-ts';

const tcpSocketOptions = TcpSocketOptions.create()
  .withHost(host)
  .withPort(port)
  .withFraming({ kind: 'length-prefixed' })
  .withTarget(subscriber);
new TcpSocketActor(tcpSocketOptions);
```

Optionaler Framer, der den Byte-Stream in diskrete Nachrichten
parst.  Das Framework liefert:

- **`LengthPrefixedFramer`** — 4-Byte- (konfigurierbares) Längen-
  Prefix.
- **`DelimitedFramer`** — zeilen-delimited oder durch eine
  bestimmte Byte-Folge abgeschlossen (Newline, NUL usw.).
- **Custom** — implementiere `TcpFramer` für Protokolle wie
  HTTP/1.1, redis-protocol usw.

Mit einem Framer enthält die `data`-Nachricht von `TcpInbound`
**einen vollständigen Frame**, nicht beliebige Chunks.

## Listener-Muster für eingehend

Das Framework liefert keinen "TCP-Listener-Actor" — schreib einen,
der pro angenommener Verbindung `TcpSocketActor`-ähnliche Handler
spawnt:

```ts
// In einem Init-Actor:
import net from 'node:net';

const server = net.createServer((socket) => {
  const handler = system.spawn(
    Props.create(() => new IncomingConnectionActor(socket)),
  );
});
server.listen(port, host);
```

Für die meisten Anwendungsfälle gilt: **bau dir keinen eigenen
TCP-Server** — nimm stattdessen HTTP, WebSocket oder ein
höherrangiges Protokoll.

## Wann TCP

Drei legitime Einsatzfälle:

1. **Mit Legacy-Protokollen sprechen** — proprietäre Protokolle,
   für die es keine höherrangigen Wrapper gibt.
2. **Eigene Binärprotokolle** — Game-Server, Metric-Collector mit
   eigenem Wire-Format.
3. **Brücken zu Nicht-HTTP-Services** — Message-Queues mit
   proprietärem Wire (z. B. einige Finanzprotokolle).

Für neue Anwendungsprotokolle gilt: **greife nicht zuerst zu raw
TCP**.  HTTP, WebSocket oder gRPC sind für fast alles die besseren
Startpunkte.

import { Aside } from '@astrojs/starlight/components';

<Aside type="caution" title="TCP gibt dir Bytes, keine Nachrichten">
  ```ts
  // 'hello' und 'world' separat senden
  tcp.tell({ kind: 'send-text', payload: 'hello' });
  tcp.tell({ kind: 'send-text', payload: 'world' });
  // Der Empfänger sieht evtl.: 'hel', 'loworld' oder 'hellowor', 'ld' — Chunks nicht aligned
  ```
  Nimm einen Framer.  Immer.  Ohne einen muss Anwendungscode
  beliebige Byte-Grenzen wieder zusammensetzen — leicht
  falsch zu machen.
</Aside>

<Aside type="caution" title="Kein Connection-Multiplexing">
  ```ts
  // Jede TCP-Verbindung ist ein Stream
  ```
  Du willst bidirektional + multiplexed?  Nimm HTTP/2 (über gRPC)
  oder WebSocket.  Raw TCP ist Single-Stream pro Verbindung.
</Aside>

<Aside type="caution" title="Backpressure ist nicht automatisch">
  ```ts
  // Schneller senden als das Netz abgeben kann
  for (let i = 0; i < 1_000_000; i++) tcp.tell({ kind: 'send', payload: huge });
  ```
  Der Kernel-Buffer füllt sich; nachfolgende Sends stauen sich im
  Speicher.  Für anhaltende Schreib-Workloads den Producer drosseln
  oder ein höherrangiges Protokoll mit eingebauter Flusskontrolle
  nutzen.
</Aside>
