Die Features des Frameworks haben unterschiedliche
Unterstützungs-Level über Runtimes hinweg. Diese Seite ist die
**definitive Referenz**.

Für jede Zeile:

- **✓** = unterstützt, in CI getestet.
- **⚠** = funktioniert, aber nicht aktiv CI'd.
- **✗** = nicht unterstützt.
- **N/A** = trifft nicht zu.

## Empfohlene Ziele

```
Für neue Projekte:                            Bun
Für Node-Shop-Umgebungen:                     Node 20+
Für Lambda / Serverless:                      Node 20+
Für Deno-First-Organisationen:                Deno (mit Einschränkungen)
Für gemischt:                                 Wähle eine pro Cluster
```

import { Aside } from '@astrojs/starlight/components';

<Aside type="caution" title="Den Warn-Marker lesen">
  ```
  ⚠ = das Framework hat keine Produktions-Stunden auf dieser Runtime
  für dieses Feature gesehen. Funktioniert meist; könnte Edge-Cases haben.
  ```
  Wenn du Probleme bei einem ⚠-Feature triffst, melde einen
  Bug — die meisten Deno-spezifischen Issues sind einfache
  Fixes.
</Aside>

<Aside type="caution" title="Version-Pinning">
  ```json
  // package.json
  { "engines": { "bun": ">=1.1", "node": ">=20" } }
  ```
  Pinne sowohl Bun- als auch Node-Engines, wenn dein Code auf
  beiden läuft — verhindert versehentlichen Dev-/Prod-Runtime-
  Drift.
</Aside>
