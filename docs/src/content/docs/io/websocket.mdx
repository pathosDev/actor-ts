---
title: WebSocketClientActor
description: Outbound WebSocket connection actor — connects to a remote WS endpoint, decodes inbound frames into typed messages, sends typed outbound messages. Reconnect with backoff, outbound buffering, and a circuit breaker via BrokerActor. Wraps the runtime's native WebSocket.
---

`WebSocketClientActor` opens a **client-side** WebSocket
connection to a remote endpoint.  Inbound frames are decoded
into typed messages you handle in `onMessage`; outbound messages
are encoded and sent (buffered while disconnected, resent after
reconnect).  For accepting **incoming** WS connections (server
side), see [server WebSocket](/io/server-websocket/).

```ts
import { WebSocketClientActor, WebSocketClientOptions, wsSend } from 'actor-ts';

type ClientMsg = { kind: 'ping'; n: number };
type ServerMsg = { kind: 'pong'; n: number } | { kind: 'event'; data: unknown };

class Feed extends WebSocketClientActor<ClientMsg, ServerMsg> {
  constructor() {
    super(WebSocketClientOptions.create<ClientMsg, ServerMsg>().withUrl('ws://localhost:8080/ws'));
  }

  override onConnected(): void {
    this.send({ kind: 'ping', n: 1 });
  }

  onMessage(msg: ServerMsg): void {
    // decoded server message
    if (msg.kind === 'pong') this.log.info(`pong ${msg.n}`);
  }
}

const feed = system.spawn(Props.create(() => new Feed()), 'feed');
```

The two type parameters read from the client's point of view:
`TOut` = the messages **this client sends**, `TIn` = the
**decoded server messages** it receives.

`WebSocketClientActor` extends the shared
[BrokerActor base](/io/broker-actor-base/), so it inherits
reconnect-with-backoff, the outbound buffer across reconnects,
and the circuit breaker for free.

## Settings

The constructor takes `Partial<WebSocketClientSettings<TOut, TIn>>`.
Only `url` is required; everything else has a default.

```ts
interface WebSocketClientSettings<TOut, TIn> extends BrokerCommonSettings {
  url:              string;                        // required — ws:// or wss://
  protocols?:       string | string[];             // subprotocols
  headers?:         Record<string, string>;        // Node / `ws` only
  codec?:           WsCodec<TOut, TIn>;             // default jsonCodec()
  maxFrameBytes?:   number;                         // default 1 MiB; oversize inbound dropped with a warning
  onInvalidMessage?: 'drop' | 'hook' | 'disconnect'; // default 'drop'
  pingIntervalMs?:  number;                         // application-level ping; default off
}
```

Because the settings extend `BrokerCommonSettings`, the
`reconnect` (`{ initialDelayMs, maxDelayMs, factor, maxAttempts }`),
`outboundBuffer`, and `circuitBreaker` blocks all apply — see the
[BrokerActor base](/io/broker-actor-base/) for their semantics.

## Sending and receiving

Inside the actor, `this.send(msg)` encodes the message via the
codec and enqueues it; while disconnected the message is buffered
and resent after reconnect.  Inbound frames are decoded and
delivered to `onMessage`.

```ts
class Chat extends WebSocketClientActor<ClientMsg, ServerMsg> {
  constructor() {
    super(WebSocketClientOptions.create<ClientMsg, ServerMsg>().withUrl('wss://chat.example.com/ws'));
  }

  override onConnected(): void {
    this.send({ kind: 'setName', name: 'alice' });   // encode + enqueue → true
  }

  onMessage(msg: ServerMsg): void {
    // one call per decoded server frame, in frame order
  }
}
```

`send(msg: TOut): boolean` returns whether the message was
accepted (it can be rejected if the outbound buffer is full).
For a raw frame — bypassing the codec — use
`sendRaw(frame: WsFrame): boolean`.

### Sending from another actor

Other actors don't call `send` directly; they push a typed send
through the ref with the `wsSend` helper:

```ts
import { wsSend } from 'actor-ts';

feed.tell(wsSend({ kind: 'ping', n: 42 }));
```

## Hooks

`onMessage(msg: TIn)` is abstract — you must implement it.  The
rest are optional overrides:

| Hook | When |
| --- | --- |
| `onConnected()` | The connection is (re)established.  Good place for a handshake / re-subscribe. |
| `onDisconnected(cause?: Error)` | The connection dropped; `cause` is set on error-driven drops. |
| `onInvalidMessage(error: WsDecodeError)` | A frame failed to decode (only when `onInvalidMessage: 'hook'`). |
| `onSelfMessage(msg: TSelf)` | Handle the optional third type parameter — messages the actor sends to itself. |

## Codecs

The codec turns `TOut` messages into wire frames and wire frames
back into `TIn` messages.  The default is `jsonCodec()`:

```ts
import { jsonCodec } from 'actor-ts';

new Feed(); // uses jsonCodec() — text frames <-> JSON

// With runtime validation (zod etc.):
class Validated extends WebSocketClientActor<ClientMsg, ServerMsg> {
  constructor() {
    super(
      WebSocketClientOptions.create<ClientMsg, ServerMsg>()
        .withUrl('wss://...')
        .withCodec(jsonCodec<ClientMsg, ServerMsg>({
          validate: (value: unknown): ServerMsg => ServerMsgSchema.parse(value),
        })),
    );
  }
  onMessage(msg: ServerMsg): void { /* already validated */ }
}
```

For binary protocols, `rawCodec()` is the escape hatch —
`TOut = TIn = WsFrame`, so you handle raw text/binary frames
yourself:

```ts
import { rawCodec, type WsFrame } from 'actor-ts';

class Binary extends WebSocketClientActor<WsFrame, WsFrame> {
  constructor() {
    super(WebSocketClientOptions.create<WsFrame, WsFrame>().withUrl('wss://...').withCodec(rawCodec()));
  }
  onMessage(frame: WsFrame): void {
    if (frame.kind === 'binary') this.handleBytes(frame.data);   // Uint8Array
  }
}
```

A `WsFrame` is `{ kind: 'text'; data: string }` or
`{ kind: 'binary'; data: Uint8Array }`.  Decode failures throw a
`WsDecodeError`; the inbound frame-size cap (`maxFrameBytes`) is
enforced on the **raw frame before decode**.

## Reconnect and buffering

An unexpected disconnect is handled by the
[BrokerActor lifecycle](/io/broker-actor-base/):

- The actor transitions to `disconnected` and fires
  `BrokerDisconnected` on the event stream.
- The reconnect cycle starts, with exponential backoff per the
  `reconnect` settings (`initialDelayMs`, `maxDelayMs`, `factor`,
  `maxAttempts`).
- After a successful re-connect, `onConnected()` runs and
  `BrokerConnected` fires.

**Outbound messages** sent while disconnected are **buffered**
per the outbound-buffer policy and resent after reconnect.  The
inherited **circuit breaker** trips after repeated connection
failures so a dead endpoint doesn't spin the reconnect loop
forever.

## Ping / keep-alive

```ts
new Feed(); // ...
new WebSocketClientActor(
  WebSocketClientOptions.create().withUrl('wss://...').withPingIntervalMs(30_000),
);
```

An application-level ping every `pingIntervalMs` keeps
intermediate proxies / load balancers from closing idle
connections — most production setups want it.  The protocol-level
WebSocket ping (control frame) is separate and handled by the
runtime; the application-level ping is for **proxies that inspect
payload, not frames**.

## HOCON config

Client defaults can be set under
`actor-ts.io.broker.websocket`:

```hocon
actor-ts.io.broker.websocket {
  url             = "wss://realtime.example.com/feed"
  protocols       = ["my-proto-v1"]
  headers         { authorization = "Bearer ..." }
  pingIntervalMs  = 30000
  maxFrameBytes   = 1048576
  reconnect       { initialDelayMs = 500, maxDelayMs = 30000, factor = 2.0, maxAttempts = -1 }
  circuitBreaker  { /* ... */ }
  outboundBuffer  { /* ... */ }
}
```

Precedence follows the broker convention: constructor argument >
HOCON > built-in defaults.

## When to use it

Three primary fits:

1. **Subscribing to real-time data feeds** — stock tickers,
   crypto exchanges, social-media streams.
2. **Outbound WebSocket clients** to third-party services
   (Slack RTM, streaming LLM APIs, IoT vendor APIs).
3. **Custom WS-based protocols** layered on top — use
   `rawCodec()` for binary wire formats.

For server-side WebSocket (accepting client connections), use
the [server WebSocket](/io/server-websocket/) page.

<Aside type="note" title="Renamed from WebSocketActor">
  This actor was previously called `WebSocketActor`.  The command
  interface (`{ kind: 'subscribe' | 'send' }`) is gone: implement
  `onMessage`, call `this.send(msg)`, and push typed sends from
  other actors with `ref.tell(wsSend(msg))`.
</Aside>

<Aside type="caution" title="`ws` peer dependency on Node < 22">
  ```bash
  npm install ws        # Node < 22 only
  ```
  Bun, Deno, and Node 22+ ship a native `WebSocket`, so no peer
  dependency is needed there.  On older Node, install the `ws`
  package.  The `headers` setting is only honoured on the Node /
  `ws` path — browsers and the native runtime `WebSocket` don't
  allow custom handshake headers.
</Aside>

<Aside type="caution" title="Browser-side WebSocket is different">
  This actor is for **server-side outgoing** connections.  Browser
  clients connecting to a server use the browser's native
  `WebSocket` API directly.
</Aside>
